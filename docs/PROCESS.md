1. What did you build?
I built a fitness studio website using Claude Code that included essential business features like a homepage, class schedule,
membership details, trainer profiles, and a contact/sign-up section. I also added a review/testimonials section to the index.html
homepage to make the site feel more realistic and build credibility with potential clients. Including reviews improves user
experience because it provides social proof, helps new visitors trust the business, and can influence decision-making when someone
is considering joining the studio. The overall goal was to create a professional-looking site that was easy to navigate and could
support user engagement and conversions.

3. How did micro-iteration feel?
Micro-iteration felt very manageable because I could focus on one small improvement at a time and immediately see the results in
the browser. It made the process less overwhelming and helped me stay organized since I wasn’t trying to fix everything at once.

5. What did self-review catch?
Self-review helped me notice things I might have missed initially, like spacing inconsistencies, navigation issues, and small
usability problems. It also helped improve the overall flow of the site by making sure buttons, links, and content made sense
from a user’s perspective.

7. Tool impressions.
The browser-based AI tools were convenient and easy to use, especially for quick development and visual changes. Being able to
see updates instantly without extra setup made the workflow feel faster and more beginner-friendly compared to traditional
coding environments.

9. Self-review patterns.
One pattern I noticed was that I needed to check responsiveness and user flow frequently after making changes. Clicking through
the site like a real user helped catch more issues than just looking at the code. I also found myself refining wording and layout
together since both affect user experience.

11. Browser tool vs. CLI comparison.
The browser tool was better for rapid prototyping, design adjustments, and testing the user experience in real time. A CLI
environment would probably be more useful for larger projects with more complex functionality, version control, or backend
integration. The browser approach feels more accessible, while CLI offers more control for advanced development.

13. When would you use micro-iteration + self-review?
I would use micro-iteration and self-review when working on frontend design, user interfaces, or projects where requirements
might change often. It’s especially helpful when learning new tools or building prototypes because it allows continuous improvement
 without needing to redo large parts of the project.
